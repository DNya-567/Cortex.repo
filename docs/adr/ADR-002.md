# ADR-002: Separate Validation from Persistence

## Status
accepted

## Context
The AuthService class initially mixed validation logic with state management. This made testing difficult and error handling unclear.

## Decision
Separate validation concerns from the persistence layer. Validators return boolean, while persistence methods handle side effects.

## Consequences
- Easier to test validation logic independently
- Clear separation of concerns
- More explicit error handling
- Slight increase in code verbosity

## Affected Files
- test-codebase/auth.js
- test-codebase/userRepository.js
